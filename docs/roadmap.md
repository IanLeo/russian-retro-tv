# Дальнейшие шаги

## Phase 0: Решения до разработки

1. Выбрать рабочее название и домен.
2. Определить юридическую позицию: только embed/link-out, без загрузки видео на свои сервера.
3. Подготовить privacy policy и terms с явным упоминанием YouTube API Services и Telegram embeds.
4. Выбрать стек: рекомендуемый старт - Next.js или Vite React + TypeScript, Supabase/Postgres для каталога, cron/worker для проверок YouTube.

## Phase 1: MVP

1. Сделать фронтенд ретро-телевизора: экран, кнопки, категории, год, fullscreen, shuffle, playlist.
2. Подключить YouTube IFrame Player API.
3. Ввести человекочитаемый каталог JSON и генератор compact catalog.
4. Сделать минимальную админку или CSV-import для ручной модерации.
5. Добавить health-check роликов через YouTube Data API.
6. Собрать 200-500 проверенных роликов по 1990-2009.

## Phase 2: Контент-пайплайн

1. Поисковый сбор кандидатов из YouTube Data API по запросам: `ОРТ 1995 реклама`, `НТВ 2001 Намедни`, `РТР 1997`, `ТВ-6 1998`, `СТС 2002`, `MTV Россия 2000`, `Муз-ТВ 1999`.
2. Извлечение ссылок из публичных Telegram-каналов через MTProto user client с ручной модерацией.
3. Очередь модерации: дедупликация, год, телеканал, категория, tags, source attribution.
4. Процедура удаления по запросу автора/правообладателя.

## Phase 3: Продукт

1. Разделить 90-е и 00-е на отдельные "телевизоры" или сделать одну сетку с переключателем десятилетия.
2. Добавить "эфирные пакеты": федеральное ТВ, региональное ТВ, реклама, музыка, ночной эфир, новости.
3. Сохранение пользовательских подборок локально.
4. Форма предложить видео.
5. Public changelog каталога.

## Ближайший спринт

1. Согласовать название репозитория и GitHub owner.
2. Развернуть скелет приложения.
3. Реализовать YouTube-player wrapper.
4. Сделать первые 30-50 seed-записей вручную из `Орбита-4`, mrcatmann/staroetv и YouTube-поиска.
5. Настроить GitHub Actions: lint, typecheck, catalog validation.
