# Исследование MyRetroTVs

Дата среза: 2026-05-18.

## Функциональность оригинала

MyRetroTVs состоит из отдельных поддоменов по десятилетиям, например `80s.myretrotvs.com`. Главная метафора - старый телевизор с экраном, кнопками, ручками и боковым меню соседних десятилетий.

Ключевые функции:

- включение/выключение телевизора;
- переключение канала вперед/назад;
- выбор года внутри десятилетия;
- фильтры категорий: Cartoons, Comedy, Commercials, Drama, Gameshows, Kids, Movies, Music, News, Other, Soaps, Specials, Sports, Talkshows, Trailers;
- режимы playlist и shuffle;
- полноэкранный режим;
- изменение громкости;
- "телевизионный шум", glow, звуковые эффекты;
- открытие текущего ролика на YouTube с текущим таймкодом;
- репорт/предложение видео через внешние формы;
- управление клавиатурой.

## Техническая модель

Оригинал - Vite/JavaScript SPA. HTML содержит готовую разметку ретро-телевизора, а логика загружается из JS-бандла. Воспроизведение сделано через YouTube IFrame API: клиент создает `YT.Player`, передает `enablejsapi: 1`, управляет `loadVideoById`, `loadPlaylist`, `seekTo`, `setVolume`, событиями `onReady`, `onError`, `onStateChange`.

Каталог хранится отдельно в публичном `d.json`. Для 80-х файл на момент проверки:

- дата каталога: `2026-04-25T23:50:10.053Z`;
- 10 массивов по годам 1980-1989;
- примерно 44 302 коротких записи;
- записи компактные: 11 символов видео-ID + 1 символ категории, при этом ID обфусцируется простой перестановкой.

Распределение записей в проверенном `80s/d.json`:

- 1980: 2788
- 1981: 3002
- 1982: 3519
- 1983: 3451
- 1984: 4465
- 1985: 5666
- 1986: 5618
- 1987: 5448
- 1988: 5192
- 1989: 5153

## Что стоит повторить

- Эффект "линейного ТВ", где пользователь не выбирает конкретное видео из поиска, а переключает поток.
- Каталог как отдельный легкий JSON, который можно деплоить независимо от фронтенда.
- Ручная/кураторская валидация дат, категорий и названий.
- Обработка ошибок YouTube: если ролик удален/заблокирован/не embeddable, автоматически переходить к следующему.
- Отдельный режим "playlist", чтобы смотреть подборку подряд.

## Что лучше изменить

- Не полагаться на клиентский API-ключ в публичном бандле для всех операций поиска.
- Не хранить только обфусцированные строки: для разработки лучше иметь человекочитаемый source-of-truth каталог, а компактный JSON генерировать на сборке.
- Добавить админку/модерацию с полями правового риска и источника.
- Сразу заложить проверку `embeddable`, доступности, возраста, региона и удаления роликов.
