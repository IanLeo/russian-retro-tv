# Russian Retro TV

Исследовательский репозиторий для сервиса в духе MyRetroTVs, но с российским телевидением 1990-х и 2000-х: передачи, реклама, заставки, новости, музыкальные фрагменты и редкие VHS-оцифровки, воспроизводимые как "переключение каналов" внутри ретро-телевизора.

## Идея

Пользователь выбирает десятилетие, год и категории контента, включает виртуальный телевизор и переключает "каналы". Видео физически не хранятся в проекте: MVP должен воспроизводить легально доступные публичные источники через официальные embed/API-механизмы, в первую очередь YouTube IFrame Player API, а Telegram использовать как источник ссылок и кураторских подборок.

## Что уже сделано

- Исследована механика MyRetroTVs: SPA, ретро-UI, категории, годы, shuffle/playlist modes, fullscreen, статический шум, управление клавиатурой и YouTube IFrame API.
- Проверена структура данных оригинала: публичный `d.json`, разбиение по годам, компактные записи по 12 символов с обфускацией ID и кодом категории.
- Собрана первичная карта контент-источников для российского ТВ 90-х/00-х.
- Зафиксированы технические и юридические ограничения YouTube и Telegram.

## Локальный запуск

```bash
python3 -m http.server 5173
```

Затем открыть `http://localhost:5173`.

## Рабочий вывод

Лучший стартовый путь: не скачивать видео и не скрейпить YouTube, а вести собственный каталог ссылок/ID с проверкой доступности, воспроизводить YouTube через официальный IFrame API, Telegram использовать как источник обнаружения и ручной/полуавтоматической модерации ссылок, а не как CDN для чужих файлов.
